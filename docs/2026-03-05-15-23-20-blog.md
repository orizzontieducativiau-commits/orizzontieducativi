# Implementazione Blog

**Data:** 2026-03-05
**Stato:** Completato

## Obiettivo

Implementare un blog completo con due viste:
1. **Pagina galleria** (`/blog`) — elenco degli articoli con card visive
2. **Pagina articolo** (`/blog/[slug]`) — contenuto completo dell'articolo caricato da file MDX

## Approccio tecnico

### Gestione articoli — MDX con frontmatter

Gli articoli saranno file `.mdx` nella cartella `content/blog/`. Ogni file contiene frontmatter YAML + contenuto Markdown con supporto immagini. MDX è scelto rispetto a `.md` puro perché:
- Supporto nativo in Next.js con `next-mdx-remote` o `@next/mdx`
- Possibilità futura di inserire componenti React nel contenuto
- Parsing del frontmatter integrato

**Struttura di un articolo:**
```mdx
---
title: "Titolo dell'articolo"
description: "Breve descrizione per la card e i meta tag"
slug: "titolo-dell-articolo"
date: "2026-03-05"
image: "/images/blog/nome-immagine.jpg"
tags: ["educazione", "australia"]
---

Contenuto Markdown dell'articolo...
```

**Libreria scelta: `gray-matter` + `next-mdx-remote`**
- `gray-matter` — parsing del frontmatter YAML dai file `.mdx`
- `next-mdx-remote` — rendering MDX lato server con Server Components di Next.js
- Nessun database necessario: i file `.mdx` fungono da "database in memoria"

### Struttura file

```
content/
  blog/
    articolo-esempio.mdx          ← articolo di esempio
src/
  lib/
    blog.ts                       ← funzioni per leggere/parsare gli articoli
  app/
    blog/
      page.tsx                    ← galleria articoli (Server Component)
      [slug]/
        page.tsx                  ← pagina singolo articolo (Server Component)
  components/
    blog/
      blog-card.tsx               ← card articolo per la galleria
      blog-header.tsx             ← header della pagina blog
      mdx-content.tsx             ← wrapper per il rendering MDX
```

### Design — Ispirato alla homepage

Il design seguirà le convenzioni visive già presenti:
- Font Caveat per le etichette decorative
- Palette colori coerente (emerald, gray, nero)
- Card con bordi sottili e ombre leggere (stile `services.tsx`)
- Layout responsive con `container mx-auto max-w-5xl`
- Blob decorativi come nella Hero section

**Pagina galleria (`/blog`):**
- Hero compatta con titolo e sottotitolo in stile Caveat
- Griglia responsive di card (1 colonna mobile, 2 tablet, 3 desktop)
- Ogni card mostra: immagine di copertina, titolo, descrizione breve, data, tag come Badge
- Hover con transizione ombra (come le card Services)

**Pagina articolo (`/blog/[slug]`):**
- Layout a colonna stretta centrata (max-w-3xl) — stile editoriale
- Immagine di copertina hero a larghezza piena
- Tipografia ottimizzata per la lettura (prose)
- Pulsante "Torna al blog"
- Metadati: data, tag, tempo di lettura stimato
